# Нормоконтроль чертежа (CHECK_FILE)

Специализированный LISP-скрипт для AutoCAD (и аналогичных CAD-систем), разработанный для автоматической проверки структуры чертежа на соответствие строгому внутреннему стандарту (шаблону) предприятия или отрасли. Скрипт сканирует таблицы данных чертежа и мгновенно выявляет любые сторонние, ошибочные или некорректные элементы.

Инструмент незаменим для инженеров-геодезистов, картографов и специалистов нормоконтроля при сдаче топографо-геодезических планов (в частности, масштаба 1:500), исполнительных схем и цифровых моделей местности в государственные или корпоративные фонды данных.

---

## Основные возможности

* **Тотальный аудит по трем базам данных:** Скрипт производит перекрестную проверку трех ключевых таблиц символов AutoCAD:
  1. **Слои (`Layers`):** Ищет слои, которых не должно быть в шаблоне, а также контролирует наличие обязательных целевых слоев.
  2. **Блоки (`Blocks`):** Проверяет имена всех описаний блоков на чертеже, отсекая системные пространства модели и листов.
  3. **Типы линий (`Linetypes`):** Выявляет несанкционированные стили линий.
* **Контроль засорения слоя «0»:** Скрипт осуществляет поштучный перебор элементов пространства модели (`ModelSpace`). Он подсчитывает количество примитивов (линий, текстов, штриховок), которые ошибочно начерчены непосредственно на слое `0`, при этом корректно игнорируя вставки блоков (`AcDbBlockReference`), для которых слой `0` является нормативным базовым слоем вложенности.
* **Контроль недостающих элементов:** Помимо поиска «мусора», скрипт вычисляет массив `missingLayers` — список слоев, которые обязаны присутствовать в структуре чертежа согласно стандарту, но были случайно удалены пользователем.
* **Вывод структурированного отчета:** Вместо загромождения экрана диалоговыми окнами, утилита выдает лаконичный и информативный текстовый лог со списком всех нарушений прямо в командную строку AutoCAD.

## Инструкция по установке и запуску

1. **Загрузка скрипта в AutoCAD:**
   * Введите в командной строке `APPLOAD` (или `ЗАГРИСП`).
   * Выберите файл `check_file.lsp` и нажмите **Загрузить** (Load).
   * При необходимости добавьте файл в «Автозагрузку» (Startup Suite).

2. **Запуск команды:**
   * Введите в командную строку: `CHECK_FILE`

3. **Анализ результатов:**
   * **Вариант 1 (Файл чист):** Если чертеж полностью соответствует стандарту, в консоли появится сообщение:
     ```text
     ======================================
     V Всё хорошо, лишних элементов нет.
     ```
   * **Вариант 2 (Обнаружены нарушения):** Если в файле присутствует мусор или не хватает слоев, скрипт выведет детальный отчет:
     ```text
     ======================================
     ПРОБЛЕМАХ В ФАЙЛЕ:
     Лишние слои: [Имена ошибочных слоев]
     Отсутствуют обязательные слои: [Имена пропавших слоев]
     Лишние блоки: [Имена несанкционированных блоков]
     Лишние стили линий: [Имена лишних типов линий]
     Слой "0": есть элементы (ХХ)
     --------------------------------------
     X Есть моменты, требующие внимания.
     ```

## Техническая информация для администратора шаблона

Для адаптации скрипта под стандарты вашей компании, вы можете открыть файл `check_file.lsp` любым текстовым редактором (например, Блокнотом) и отредактировать списки допустимых элементов в соответствующих блоках кода:
* `allowedLayers` — список разрешенных имен слоев.
* `allowedBlocks` — список разрешенных условных знаков (блоков).
* `allowedLTypes` — список разрешенных типов линий.
